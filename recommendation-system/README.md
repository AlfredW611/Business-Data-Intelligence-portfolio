# Recommendation System Based on User Purchase History

## Business Problem
How can an e-commerce platform recommend relevant products to users 
based on their historical purchase behavior?

## Dataset
- Public e-commerce transaction dataset
- Over 50,000 user-item interactions

## Approach
- Data cleaning and preprocessing
- User-item matrix construction
- Collaborative filtering (User-based)

## Models
- Cosine similarity
- Top-N recommendation

## Results
- Generated personalized product recommendations
- Improved relevance compared to popularity baseline

## Business Insights
- Repeat customers show strong category-level preferences
- Recommendations can support targeted promotions

## Limitations & Next Steps
- Cold-start problem not addressed
- Future work: matrix factorization or deep learning
